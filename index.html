AI 不只是会聊天，还能帮我们一步步把事情做完

我来自技术侧数据部门，也就是 UData 所在的部门。今天想用最通俗的方式，和大家聊聊几个爆火的 AI 概念，以及它们能帮我们解决哪些工作问题。

从“我想知道什么”到“最后给出结果”，中间的做法可以反复复用

分享大纲

  • 1）先讲清楚大模型、Agent、Skill 到底分别是什么
  • 2）再看 AI 最适合先解决哪些业务问题，重点看两个真实场景
  • 3）最后看 UData-AI 后面准备怎么一步步推进

核心原理：大模型、Agent、Skill 分别做什么？

先只记住一句话：大模型像大脑，Agent 像手脚，Skill 像我们平时积累下来的 SOP。

1. 大模型到底是什么？

可以把它理解成读过大量公开资料后，能根据输入快速组织回答的“超级补全器”

不同模型，可以理解成不同类型的大脑

  • GPT：整体比较均衡，适合问答、写作、整理思路这类通用任务。
  • Claude：长文理解和代码能力比较强，复杂任务里比较常见。
  • Gemini：图文音视频一起处理的能力更强，适合多模态场景。
  • DeepSeek：成本比较友好，适合高频调用和批量处理。
  • Qwen：中文能力和开源生态比较活跃，国内很多场景用得多。

2. Agent 和 Skill 为什么会出现？

Agent

大模型很会“说”，但不一定真的会“干”。Agent 更像一个数字员工，您给它目标，它会自己拆步骤、做动作、看结果。

一句话：从只会聊天，变成真的能办事。

Skill

Agent 如果没有方法，做事就容易自由发挥。Skill 就像工作里的 SOP，把常见任务的做法沉淀下来，后面就能稳定复用。

例如：算对比、找原因、整理字段、自动出报告。

为什么 Agent 会火？

因为大家发现，光来回对话还不够，很多工作其实更需要“直接帮我做完”。

比如：写文档、取数据、发消息、建日程、整理结论，这些都更像“干活”，不是单纯聊天。

现在大家常说的数字员工，本质上讲的就是这件事。

为什么 Skill 很重要？

  • 不给指引的 Agent，很像刚来的实习生，容易自由发挥。
  • 有了 Skill，做法就能沉淀下来，后面谁来做都更稳。
  • 大家工作里的 SOP、模版、经验，其实都很适合沉淀成 Skill。
  • 现在很火的机器人、本体智能，本质上也是把这套搬到了现实世界。

3. 它们怎么一起工作？

先由大模型听懂问题，再由 Agent 拆步骤、接系统、做执行，最后按 Skill 的做法把结果交回来。

这套逻辑怎么记？

总结来说：大模型是大脑，Agent 是手脚，Skill 是后天学会的做事方法。现在这个领域的发展，说白了就是让大脑更聪明、手脚更稳、做事方法更成熟。

业务案例：哪些事最适合先做？

优先挑那些“经常做、很费时间、做法又比较固定”的事，这样最容易先看到效果。比如场地指标没达成时，大家经常就得解释原因，这类问题就特别适合先做。

AI分析业务应用案例

Skill 封装：再把这些做法整理成可复用的技能，后面遇到类似任务，就可以直接调用。

统一方法：先把这些高频场景里共通的做法整理出来，后面遇到类似问题就能更快复用。

1 丢损波动归因

这类问题以前主要靠人一条条看，既慢也累。现在可以先由系统帮忙找重点，再由人工做最后确认。

好处：把大量重复筛查的时间省下来。

2 真实场景 1：分拣异常归因（未达成明细）

以前需要先拼很多字段，再人工判断原因，耗时长，口径也容易不一致。现在可以先自动定位未达成运单，再补出一部分原因，人工只做最后复核。

3 真实场景 2：成本诊断（红黑榜分析）

以前场地指标没达成时，大家往往要到处找数据、拼数据、解释原因，不同人分析出来的结论还可能不一样。现在系统可以先找出成本靠后的区域和场地，再往下看问题主要出在哪些环节，并给出改善建议。

效果：深度分析报告的整理时间能明显缩短，结论也更统一。

4 通用分析与预测场景

除了归因，还可以用在预测、做报表、找问题等场景。做得多了以后，就能慢慢积累成一套常用工具箱。

后面准备怎么一步步推进？

不是一次做完，而是分三步推进：先把基础打牢，再逐步扩能力，最后做到更主动

规划中的产品形态后续不只是把能力做出来，也要把入口、交互和使用方式做得更顺手。下面这几张图，分别对应我们想推进的几个方向。

交互方式：先把统一入口和对话式交互做好，让用户一进来就知道怎么提问、怎么选能力、怎么发起任务。

技能广场：把沉淀下来的能力做成可复用的技能，后面谁有类似问题，都可以直接挑一个来用。

智能报告：把报告生成过程产品化，不只是出一段结论，还能按固定框架持续补全内容，方便业务直接拿去用。

智能问数：围绕“我想快速知道一个结论”这种需求，做到边查边解释，让用户更快拿到图表和答案。

  • Q2（先打基础）：先把常用做法整理出来，先跑通问数和报告这两个最常见的场景。
  • Q3（再扩场景）：把找数、问数、报告、搭建这些能力做得更完整，也让更多人能自己配置使用。
  • Q4（更进一步）：希望以后它不只是等人来问，而是能主动发现异常、主动提醒、主动给分析建议。

目标说得更简单一点，就是让更多人都能更轻松地用数据，把找数、做数、看数这件事整体提速。
